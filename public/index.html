GitHub Repository Quick Start

What It Does

release-gate sits between testing and deployment, validating agents against 4 governance checks:

💰 ACTION_BUDGET

Prevent cost explosions with automatic cost estimation and daily limits.

📋 INPUT_CONTRACT

Validate request schemas are defined and tested with valid/invalid samples.

⏹ FALLBACK_DECLARED

Ensure kill switches, fallback modes, and ownership exist.

🔐 IDENTITY_BOUNDARY

Enforce authentication, rate limits, and data isolation.

Install & Try (5 minutes)

$ pip install release-gate $ cat > governance.yaml << EOF project: name: my-agent agent: model: gpt-4-turbo daily_requests: 500 checks: action_budget: max_daily_cost: 100 EOF $ release-gate run governance.yaml
