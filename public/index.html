The 4 Governance Checks

✓ INPUT_CONTRACT

Validate request schemas. Ensure valid and invalid samples pass/fail correctly. No ambiguous APIs.

✓ FALLBACK_DECLARED

Kill switches, fallback modes, ownership assigned, runbook URLs. Operational safety proven.

✓ IDENTITY_BOUNDARY

Authentication required, rate limits configured, data isolation defined. Access control enforced.

✓ ACTION_BUDGET

Token limits, retry counts, cost caps, concurrency guards. Resource constraints proven.

Quick Start (5 Minutes)

1. Install

No external services. No data transmission. Local-first evaluation.

pip install pyyaml jsonschema

2. Initialize

Creates a sample configuration template with all 4 checks.

python cli.py init --project my-agent

3. Configure

Define governance policies in simple YAML. No code required.

vim release-gate.yaml # Add your checks

4. Validate

Gate validates your agent configuration. Exit code tells CI/CD what to do.

python cli.py run \ --config release-gate.yaml

5. Gate Decision

PASS (0), WARN (10), or FAIL (1). Your CI/CD responds accordingly.

if $? -eq 0: deploy elif $? -eq 10: manual_review else: block_deployment

6. Audit Trail

JSON report shows what passed, what warned, what failed, and why.

cat readiness_report.json # Full compliance evidence

GitHub Actions Integration

One-Command Deployment Gate

Add this to your workflow. release-gate runs on every pull request, gates your deployments.

- name: Governance Gate run: | pip install pyyaml jsonschema python cli.py run \ --config release-gate.yaml \ --format json \ --output report.json - name: Parse Decision run: | EXIT_CODE=$? if [ $EXIT_CODE -eq 0 ]; then echo "✓ PASS - Deploying" echo "GATE_DECISION=PASS" >> $GITHUB_ENV elif [ $EXIT_CODE -eq 10 ]; then echo "⚠ WARN - Manual review needed" echo "GATE_DECISION=WARN" >> $GITHUB_ENV else echo "✗ FAIL - Deployment blocked" echo "GATE_DECISION=FAIL" >> $GITHUB_ENV exit 1 fi - name: Upload Report uses: actions/upload-artifact@v3 with: name: governance-report path: report.json
