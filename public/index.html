How It Actually Works

Three steps from "maybe it's ready" to "definitely ready"

1

Define Your Governance Policies

Create a YAML file that declares your safety requirements: request contracts, kill switches, ownership, fallback modes, budget limits, and more.

python cli.py init --project my-agent
↓
2

Run the Governance Check

Execute release-gate against your configuration. It validates every policy, tests samples, and generates evidence of compliance.

python cli.py run --config release-gate.yaml
↓
3

Get Your Answer: PASS | WARN | FAIL

release-gate returns a decision with exit codes (0, 10, 1) perfect for CI/CD pipelines. Then you deploy or fix issues.

if [ $? -eq 0 ]; then deploy; fi

What Gets Enforced

Comprehensive governance checks across all dimensions

📋

Request Contracts

Validate request schemas. Test valid samples pass. Test invalid samples fail. Ensure well-defined interfaces.

🔧

Operational Safeguards

Enforce kill switches, fallback modes, ownership assignments, and runbook documentation. Be ready for emergencies.

🔐

Identity & Access

Verify authentication requirements, rate limits, and data isolation boundaries. Control who can use your agent.

💰

Resource Budgets

Enforce token limits, retry constraints, and cost caps. Prevent runaway expenses before they happen.

🎯

Policy Enforcement

Convert safety intentions into actual checks that block bad deployments. Governance that actually works.

📊

Audit Evidence

Generate machine-readable compliance reports. Perfect for auditors. SOC2, ISO, HIPAA ready.
