How it works

Three steps to safe deployment

release-gate slots into your existing CI/CD pipeline. No backend, no dashboard, no sign-up.

1

Write a governance.yaml next to your code

Declare your model, expected usage, daily budget cap, fallback owner, and kill switch. Takes about 5 minutes — or use release-gate init for an interactive wizard.

governance.yaml
# Tell release-gate about your agent agent: model: gpt-4-turbo daily_requests: 5000 checks: action_budget: max_daily_cost: 500 # block if exceeded fallback_declared: team_owner: platform-team kill_switch: type: feature-flag
2

Run the Impact Simulator before every deploy

One command shows your normal cost, the runaway-loop worst case, and exactly what money is at risk if a safeguard is missing. Engineering leaders see dollars, not YAML warnings.

$ release-gate impact governance.yaml Scenario Daily Monthly ————————————————————————————— Normal operation $132.00 $3,960.00 Runaway loop $5,000.00 $150,000.00 ————————————————————————————— Risk delta $4,868.00 $146,040.00 No governance gaps detected. FINAL VERDICT: ✓ APPROVED
3

Gate every push automatically in CI

Add release-gate to your GitHub Actions workflow. Exit code 0 = deploy. Exit code 1 = blocked. The HTML report is uploaded as a CI artifact — your team sees the full cost breakdown without leaving GitHub.

.github/workflows/governance.yml
- name: AI Agent Governance Gate uses: VamsiSudhakaran1/release-gate@v0.5.0 with: command: impact html-report: report.html # → CI artifact

CI/CD Integration

5 lines in GitHub Actions

Works with GitHub Actions, GitLab CI, Jenkins, and any shell. The HTML report is automatically uploaded as a CI artifact.

# .github/workflows/governance.yml name: AI Agent Governance on: [push, pull_request] jobs: governance: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Impact check uses: VamsiSudhakaran1/release-gate@v0.5.0 with: command: impact html-report: report.html # report is auto-uploaded as CI artifact
📊

HTML report as CI artifact

Every PR gets an Impact Simulator report — cost projections, risk delta, governance gaps — without leaving GitHub.

⚙️

Exit codes your pipeline understands

0 = PASS (deploy), 10 = WARN (review needed), 1 = FAIL (blocked).

🔒

Cryptographic locking (v0.5)

Sign governance.yaml with RSA-PSS + SHA256. Verify in CI that no one changed limits post-review.

Governance checks

5 checks. One decision.

Each check maps to a real failure mode — cost explosion, no kill switch, open access, bad inputs.

CheckWhat it validatesBlocked when
ACTION_BUDGET Estimated daily cost vs. declared budget cap Cost exceeds max_daily_cost or no budget set
BUDGET_SIMULATION Projected cost with retries, caching & spike multipliers across 10+ models Projected cost exceeds budget or multipliers are out of range
FALLBACK_DECLARED Kill switch, fallback mode, team owner, runbook URL Any field missing — no owner means no one gets paged at 3 AM
IDENTITY_BOUNDARY Auth required, rate limit configured, data isolation rules Auth is optional or rate limit absent — anyone can exhaust budget
INPUT_CONTRACT JSON Schema defined, valid & invalid sample payloads provided Schema missing (FAIL) or no valid samples (WARN)
